## 3.X Quality-Aware Weighted Anchor Fusion

### 3.X.1 Problem Definition

在 3AMVC 中，首先对每个视图独立执行 HBNC，以自动获得该视图的 anchor set 及其 anchor graph；随后依据 Eq. (8) 计算各视图的 anchor quality，并选择质量最高的视图作为 baseline view；再利用 Eq. (9) 将其余视图的 anchor graph 对齐到 baseline view，最后通过 Eq. (10) 进行等权融合。

3664647.3681273

3664647.3681273

然而，Eq. (10) 中对所有视图采用相同权重

$$Z_{\text{Aligned}}=\frac{1}{v}\Big(Z_b+\sum_{i=1,i\neq b}^{v}Z_iP_i\Big)$$

隐含假设“所有视图对最终共识图的贡献相同”。这一假设在多视图场景中往往过强，因为不同视图的 anchor representativeness 并不一致，而 3AMVC 本身已经通过 Eq. (8) 明确度量了这种差异，并用其选择 baseline view。

3664647.3681273

基于此，我们首先提出一种**仅基于视图质量的加权融合策略**，使得 anchor quality 更高的视图在最终融合中占更大比重。

---

### 3.X.2 Quality-Aware Weighted Fusion Formulation

设共有 $v$ 个视图，baseline view 为 $b$。对齐后的 anchor graph 记为

$$\widetilde Z_b = Z_b,\qquad  
\widetilde Z_i = Z_iP_i,\quad i\neq b,$$

其中 $Z_i\in\mathbb{R}^{n\times m_i}$，$P_i\in\{0,1\}^{m_i\times m_b}$，因此对齐后 $\widetilde Z_i\in\mathbb{R}^{n\times m_b}$，与 baseline view 的 anchor graph 具有一致维度，可直接进行融合。该记号与 3AMVC 的 Eq. (9)(10) 保持一致。

3664647.3681273

进一步地，令第 $i$ 个视图的质量分数定义为

$$q_i=\sum_{c=1}^{m_i}\sum_{x_j\in\Theta_c^{(i)}}\|x_j-\theta_c^{(i)}\|_2^2,  
\tag{11}$$

其中 $\Theta_c^{(i)}$ 表示第 $i$ 个视图下第 $c$ 个 anchor 所对应的样本集合，$\theta_c^{(i)}$ 为对应 anchor。该式本质上就是 3AMVC 原文 Eq. (8) 的单视图版本；原文正是使用这一量来衡量 anchor representativeness，并据此选择 baseline view。

3664647.3681273

3664647.3681273

由于 $q_i$ 越小表示第 $i$ 个视图的 anchor quality 越高，因此可定义质量感知权重为

$$\alpha_i=\frac{\exp(-q_i/\tau_q)}{\sum_{j=1}^{v}\exp(-q_j/\tau_q)},  
\qquad  
\alpha_i\ge 0,\quad \sum_{i=1}^{v}\alpha_i=1,  
\tag{12}$$

其中 $\tau_q>0$ 为温度参数，用于控制权重分布的尖锐程度。

最终的加权融合 anchor graph 定义为

$$Z_{\text{QW}}=\sum_{i=1}^{v}\alpha_i\widetilde Z_i.  
\tag{13}$$

为了与 3AMVC 中 Eq. (2) 的 weighted fusion 形式保持一致，也可将上式理解为在满足 simplex 约束下，对齐后 anchor graph 的带权共识估计。Eq. (2) 已经说明权重系数 $\beta$ 用于衡量各视图对 fused anchor graph 的影响，因此这里的 $\alpha_i$ 只是将该思想从“同尺寸未对齐 anchor graph”迁移到“已对齐 anchor graph”上。

3664647.3681273

---

### 3.X.3 Variable Description

式 (11)–(13) 中各变量含义如下：

* $v$：视图总数。
* $b$：baseline view 的索引，由原始 3AMVC 的质量评价策略确定。
* $n$：样本数。
* $m_i$：第 $i$ 个视图的 anchor 数。
* $m_b$：baseline view 的 anchor 数。
* $Z_i\in\mathbb{R}^{n\times m_i}$：第 $i$ 个视图的原始 anchor graph。
* $P_i\in\{0,1\}^{m_i\times m_b}$：第 $i$ 个视图到 baseline view 的匹配矩阵。
* $\widetilde Z_i\in\mathbb{R}^{n\times m_b}$：第 $i$ 个视图对齐后的 anchor graph。
* $\Theta_c^{(i)}$：第 $i$ 个视图中第 $c$ 个 anchor 的样本簇。
* $\theta_c^{(i)}$：第 $i$ 个视图中第 $c$ 个 anchor 的中心。
* $q_i$：第 $i$ 个视图的 anchor quality loss，越小越好。
* $\alpha_i$：第 $i$ 个视图的融合权重。
* $\tau_q$：质量权重温度参数。
* $Z_{\text{QW}}$：质量感知加权融合后的共识 anchor graph。

---

### 3.X.4 Optimization and Implementation

该方案不改变 3AMVC 中 HBNC、baseline selection 与 alignment 的求解流程，仅在 Eq. (10) 的融合阶段增加一步权重计算。因此整个过程可以写为：

1. 对每个视图执行 HBNC，获得 $\Theta_i$、$Z_i$ 与 $q_i$。
2. 依据最小 $q_i$ 选取 baseline view $b$。
3. 对每个 $i\neq b$，通过 Eq. (9) 求解匹配矩阵 $P_i$，并得到 $\widetilde Z_i=Z_iP_i$。
4. 根据 Eq. (12) 计算各视图权重 $\alpha_i$。
5. 利用 Eq. (13) 得到融合图 $Z_{\text{QW}}$。
6. 对 $Z_{\text{QW}}$ 进行奇异值分解，并对左奇异向量执行 K-means，得到最终聚类结果。

---

### 3.X.5 Pseudocode

**Algorithm 3** Quality-Aware Weighted Anchor Fusion

**Input:** multi-view data $\{X_i\}_{i=1}^{v}$, regularization parameter $\lambda$, temperature $\tau_q$

**Output:** fused anchor graph $Z_{\text{QW}}$, clustering labels $y$

1. **for** $i=1$ to $v$ **do**
2. $\quad$ Run HBNC on $X_i$ to obtain anchors $\Theta_i$, anchor graph $Z_i$, and quality score $q_i$
3. **end for**
4. Select baseline index

$$b=\arg\min_i q_i$$

5. Set $\widetilde Z_b=Z_b$
6. **for** $i=1$ to $v$, $i\neq b$ **do**
7. $\quad$ Solve Eq. (9) to obtain matching matrix $P_i$
8. $\quad$ Compute aligned anchor graph $\widetilde Z_i=Z_iP_i$
9. **end for**
10. Compute weights

$$\alpha_i=\frac{\exp(-q_i/\tau_q)}{\sum_{j=1}^{v}\exp(-q_j/\tau_q)}$$

11. Fuse anchor graphs

$$Z_{\text{QW}}=\sum_{i=1}^{v}\alpha_i\widetilde Z_i$$

12. Perform SVD on $Z_{\text{QW}}$
13. Apply K-means to the left singular vectors
14. Return $Z_{\text{QW}}$ and clustering labels $y$

---

### 3.X.6 Complexity Analysis

3AMVC 原文指出：HBNC 单视图复杂度为 $O(nd)$；在对齐阶段，更新 $\{Z_i\}$ 的代价为 $O(nm_id_i)$，更新 $\{\Theta_i\}$ 的代价为 $O(nm_im_b+m_im_bd)$，alignment module 的代价为 $O(m_b^2m_i)$，最终 SVD 的代价为 $O(nm_b^2)$。因此，原始 3AMVC 的总复杂度近似为 $O(n)$。

对本文方案而言，新增的计算仅包含：

1. 权重归一化计算：$O(v)$；
2. 带权求和融合：$O(vnm_b)$。

因此，质量加权融合带来的额外时间复杂度为

$$O(v)+O(vnm_b)\approx O(vnm_b),  
\tag{14}$$

不改变整个方法关于样本数 $n$ 的主导复杂度阶数。额外空间复杂度仅为存储权重向量 $\alpha\in\mathbb{R}^{v}$，即

\tag{15}$$

### 3.X.7 Hyper-parameter Discussion

该方案相较原始 3AMVC 仅新增一个主超参数 $\tau_q$。

* 当 $\tau_q$ 较小时，权重分布更尖锐，模型更倾向于依赖少数高质量视图；
* 当 $\tau_q$ 较大时，权重更平滑，模型更接近原始等权融合。

结合 3AMVC 原文对 $\beta,\lambda$ 的设置方式，可将 $\tau_q$ 在一个较小范围内网格搜索，例如

$$\tau_q\in\{10^{-2},10^{-1},1,10\}.  
\tag{16}$$

原始 3AMVC 仅涉及 $\beta$ 与 $\lambda$ 两个主要超参数，因此该方案的额外调参成本较低。

## 3.Y Quality-and-Alignment Aware Weighted Anchor Fusion

### 3.Y.1 Motivation

尽管上节方法能根据单视图 anchor quality 自适应分配融合权重，但它仍忽略了一个关键问题：**某些视图虽然自身 representativeness 较高，但其与 baseline view 的对齐结果可能并不理想**。而 3AMVC 的 Eq. (9) 明确表明，跨视图对齐同时依赖样本–anchor 关系一致性与 anchor 内部结构一致性。若某视图的对齐残差较大，则其对最终融合的有效贡献应当被抑制。

3664647.3681273

因此，我们进一步提出一种**同时考虑视图质量与对齐质量的联合加权策略**。

---

### 3.Y.2 Alignment-Aware Quality Score

对于非 baseline view $i\neq b$，根据 3AMVC 的 Eq. (9)，其对齐目标为

$$\min_{P_i}\ \|Z_b-Z_iP_i\|_F^2+\lambda\|G_b-P_i^\top G_iP_i\|_F^2.  
\tag{17}$$

其中第一项刻画 feature-level correspondence，第二项刻画 structure-level correspondence，二者共同决定第 $i$ 个视图对齐到 baseline view 的质量。该形式与原文完全一致。

3664647.3681273

据此，我们定义第 $i$ 个视图的对齐残差为

$$e_i=  
\frac{\|Z_b-Z_iP_i\|_F^2}{\|Z_b\|_F^2}  
+  
\lambda  
\frac{\|G_b-P_i^\top G_iP_i\|_F^2}{\|G_b\|_F^2},  
\qquad i\neq b,  
\tag{18}$$

并令 baseline view 的对齐残差为

$$e_b=0.  
\tag{19}$$

为了消除不同量纲和尺度的影响，分别对 $q_i$ 与 $e_i$ 做归一化：

$$\bar q_i=\frac{q_i}{\frac{1}{v}\sum_{j=1}^{v}q_j+\varepsilon},  
\qquad  
\bar e_i=\frac{e_i}{\frac{1}{v}\sum_{j=1}^{v}e_j+\varepsilon},  
\tag{20}$$

其中 $\varepsilon>0$ 为数值稳定项。

随后构造联合评分

$$s_i=\rho\bar q_i+(1-\rho)\bar e_i,  
\qquad 0\le \rho\le 1,  
\tag{21}$$

其中 $\rho$ 用于平衡单视图内部质量与跨视图对齐质量。

最后定义联合权重为

$$\alpha_i=  
\frac{\exp(-s_i/\tau_s)}{\sum_{j=1}^{v}\exp(-s_j/\tau_s)},  
\qquad  
\alpha_i\ge 0,\quad \sum_{i=1}^{v}\alpha_i=1,  
\tag{22}$$

其中 $\tau_s>0$ 为联合权重温度参数。

于是最终融合图可写为

$$Z_{\text{QAW}}=\sum_{i=1}^{v}\alpha_i\widetilde Z_i.  
\tag{23}$$

---

### 3.Y.3 Variable Description

式 (17)–(23) 中新增变量说明如下：

* $G_b,G_i$：baseline view 与第 $i$ 个视图的内部结构图，由 3AMVC 原始对齐模块使用。
* $e_i$：第 $i$ 个视图相对于 baseline view 的归一化对齐残差，越小越好。
* $\bar q_i$：归一化后的视图质量分数。
* $\bar e_i$：归一化后的对齐质量分数。
* $s_i$：联合评分，越小表示该视图越应在融合中获得更高权重。
* $\rho$：质量与对齐之间的平衡系数。
* $\tau_s$：联合权重温度参数。
* $\varepsilon$：数值稳定项。
* $Z_{\text{QAW}}$：质量–对齐联合加权后的共识 anchor graph。

---

### 3.Y.4 Optimization and Interpretation

式 (21)–(23) 的核心思想是：

* $q_i$ 衡量“该视图自身的 anchor 是否足够 representative”；
* $e_i$ 衡量“该视图与 baseline view 是否足够可对齐”。

因此，$s_i$ 同时编码了**内部可靠性**与**外部一致性**。  
当某一视图具有较小的 $q_i$ 且较小的 $e_i$ 时，其 $\alpha_i$ 会自然变大；反之，若某视图虽然内部 anchor quality 较高，但对齐残差偏大，则其贡献会被抑制，这比仅依据 $q_i$ 赋权更符合 align-then-fuse 的实际需求。

---

### 3.Y.5 Pseudocode

**Algorithm 4** Quality-and-Alignment Aware Weighted Anchor Fusion

**Input:** multi-view data $\{X_i\}_{i=1}^{v}$, regularization parameter $\lambda$, balance coefficient $\rho$, temperature $\tau_s$, stability constant $\varepsilon$

**Output:** fused anchor graph $Z_{\text{QAW}}$, clustering labels $y$

1. **for** $i=1$ to $v$ **do**
2. $\quad$ Run HBNC on $X_i$ to obtain $\Theta_i$, $Z_i$, and $q_i$
3. **end for**
4. Select baseline index

$$b=\arg\min_i q_i$$

5. Set $\widetilde Z_b=Z_b$, $e_b=0$
6. **for** $i=1$ to $v$, $i\neq b$ **do**
7. $\quad$ Solve Eq. (9) to obtain $P_i$
8. $\quad$ Compute aligned graph $\widetilde Z_i=Z_iP_i$
9. $\quad$ Compute alignment residual

$$e_i=  
\frac{\|Z_b-Z_iP_i\|_F^2}{\|Z_b\|_F^2}  
+  
\lambda  
\frac{\|G_b-P_i^\top G_iP_i\|_F^2}{\|G_b\|_F^2}$$

10. **end for**
11. Normalize $q_i$ and $e_i$ via Eq. (20)
12. Compute joint score

$$s_i=\rho\bar q_i+(1-\rho)\bar e_i$$

13. Compute weights

$$\alpha_i=\frac{\exp(-s_i/\tau_s)}{\sum_{j=1}^{v}\exp(-s_j/\tau_s)}$$

14. Fuse anchor graphs

$$Z_{\text{QAW}}=\sum_{i=1}^{v}\alpha_i\widetilde Z_i$$

15. Perform SVD on $Z_{\text{QAW}}$
16. Apply K-means to the left singular vectors
17. Return $Z_{\text{QAW}}$ and clustering labels $y$

### 3.Y.6 Complexity Analysis

与方案一相比，方案二额外增加了对齐残差评估的计算。由于 Eq. (9) 的匹配矩阵 $P_i$ 已在原始 3AMVC 的 alignment 阶段求得，因此新增开销主要来自式 (18) 中两项残差的计算。

对每个非 baseline 视图：

* 计算 $\|Z_b-Z_iP_i\|_F^2$ 的代价约为 $O(nm_im_b)$；
* 计算 $\|G_b-P_i^\top G_iP_i\|_F^2$ 的代价约为 $O(m_im_b^2)$。

因此，额外时间复杂度可近似写为

$$O\Big(\sum_{i\neq b}(nm_im_b+m_im_b^2)\Big)+O(vnm_b).  
\tag{24}$$

其中最后一项 $O(vnm_b)$ 对应带权融合本身。

在大规模场景下，由于原始 3AMVC 的主导复杂度已包含 $O(nm_id_i)$ 的 anchor graph 更新以及 $O(nm_b^2)$ 的 SVD，因此该方案虽然比方案一略贵，但通常仍不会改变整体关于 $n$ 的近线性主导趋势。

额外空间复杂度主要来自存储

$$\alpha\in\mathbb{R}^{v},\quad e\in\mathbb{R}^{v},\quad s\in\mathbb{R}^{v},$$

$$O(v).  
\tag{25}$$

### 3.Y.7 Hyper-parameter Discussion

该方案相较原始 3AMVC 新增三个参数：$\rho,\tau_s,\varepsilon$。

其中：

* $\rho$ 控制视图质量与对齐质量的相对重要性；
* $\tau_s$ 控制联合权重分布的平滑程度；
* $\varepsilon$ 仅用于避免分母为零，通常固定为 $10^{-8}$ 或 $10^{-10}$。

推荐参数范围可设为

$$\rho\in\{0.3,0.4,0.5,0.6,0.7\},  
\qquad  
\tau_s\in\{10^{-2},10^{-1},1,10\},  
\qquad  
\varepsilon=10^{-8}.  
\tag{26}$$

若希望在融合阶段更强调“可融合性”，可取较小的 $\rho$，例如 $\rho=0.3\sim 0.4$；若更强调“单视图自身质量”，则可取 $\rho=0.5\sim 0.7$。

## 3.Z Discussion and Practical Recommendation

从实现角度看，两种方案都不改变 3AMVC 的主框架，仅替换 Eq. (10) 的等权平均，因此都具有良好的可插拔性。

3664647.3681273

* **方案一** 的优点是简单、稳定、额外计算代价极低，仅新增一个主超参数 $\tau_q$，适合作为最小改动版。
* **方案二** 的优点是更符合 3AMVC 的“先评估质量，再对齐，再融合”的整体逻辑，因为它同时考虑了视图本身的 representativeness 与对齐后的可融合性；缺点是比方案一多一个平衡参数 $\rho$，并额外增加对齐残差评估成本。

若目标是先做一版稳妥实验，建议采用方案一；  
若目标是形成更完整的论文改进点，建议采用方案二作为主方法，方案一作为消融对照。
